# Phase-Oriented Structure

  - API app
  - core config/logging/error models
- `phase2/`: data ingestion, cleaning, and canonical modeling
  - ingestion pipeline
  - normalized data models
  - ingestion runner
- `phase3/`: user input and preference modeling (reserved)
- `phase4/`: retrieval and deterministic scoring (reserved)
- `phase5/`: LLM prompting and ranking (reserved)
- `phase6/`: API orchestration and resilience (reserved)
